Denizcilik uyumluluk (compliance) yönetim modülü ekle. ISM Code, ISPS Code ve MLC 2006 uyumluluğunu takip et.

Yapılacaklar:

1. shared/schema.ts dosyasına ekle:

complianceChecklists tablosu:
- id (serial PK)
- vesselId (integer, FK vessels, nullable) — gemi bazlı
- organizationId (integer, FK organizations, nullable)
- userId (varchar, FK users, not null)
- standardCode (text, not null) — "ISM", "ISPS", "MLC", "MARPOL", "SOLAS"
- standardName (text, not null)
- version (text) — "2024" gibi
- totalItems (integer, not null)
- completedItems (integer, default 0)
- compliancePercentage (real, default 0)
- status (text, default "in_progress") — "not_started", "in_progress", "compliant", "non_compliant", "expired"
- lastAuditDate (timestamp)
- nextAuditDate (timestamp)
- auditorName (text)
- notes (text)
- createdAt (timestamp, default now)

complianceItems tablosu:
- id (serial PK)
- checklistId (integer, FK complianceChecklists, not null, onDelete cascade)
- sectionNumber (text) — "1.1", "1.2", "2.1" gibi
- sectionTitle (text, not null)
- requirement (text, not null) — gereklilik açıklaması
- isCompliant (boolean, default false)
- evidence (text, nullable) — uyumluluk kanıtı açıklaması
- evidenceFileUrl (text, nullable) — kanıt dosyası
- responsiblePerson (text, nullable)
- dueDate (timestamp, nullable)
- completedDate (timestamp, nullable)
- findingType (text, nullable) — "none", "observation", "non_conformity", "major_non_conformity"
- correctiveAction (text, nullable)
- correctiveActionDueDate (timestamp)
- correctiveActionStatus (text, default "open") — "open", "in_progress", "closed", "verified"
- notes (text)
- updatedAt (timestamp, default now)

complianceAudits tablosu:
- id (serial PK)
- checklistId (integer, FK complianceChecklists, not null)
- auditType (text, not null) — "internal", "external", "flag_state", "port_state", "class"
- auditorName (text, not null)
- auditorOrganization (text)
- auditDate (timestamp, not null)
- findings (jsonb) — [{itemId, findingType, description, correctiveAction}]
- overallResult (text) — "satisfactory", "observations", "non_conformities", "detained"
- reportFileUrl (text, nullable)
- nextAuditDate (timestamp)
- notes (text)
- createdAt (timestamp, default now)

2. Standart ISM Code checklist seed data:

ISM Code (International Safety Management):
Section 1: General (1.1 Definitions, 1.2 Objectives, 1.3 Application, 1.4 Functional requirements)
Section 2: Safety and Environmental Protection Policy
Section 3: Company Responsibilities and Authority
Section 4: Designated Person(s)
Section 5: Master's Responsibility and Authority
Section 6: Resources and Personnel
Section 7: Shipboard Operations
Section 8: Emergency Preparedness
Section 9: Non-conformities, Accidents and Hazardous Occurrences
Section 10: Maintenance of Ship and Equipment
Section 11: Documentation
Section 12: Company Verification, Review and Evaluation
Section 13: Certification and Periodical Verification

Her section altında 3-5 kontrol maddesi (toplam ~40 madde)

ISPS Code (International Ship and Port Facility Security):
Part A: Mandatory Requirements
- Ship Security Assessment
- Ship Security Plan
- Ship Security Officer
- Company Security Officer
- Security Levels (1, 2, 3)
- Declaration of Security
- Security Equipment

MLC 2006 (Maritime Labour Convention):
- Title 1: Minimum requirements for seafarers
- Title 2: Conditions of employment
- Title 3: Accommodation, recreation, food and catering
- Title 4: Health protection, medical care, welfare and social security
- Title 5: Compliance and enforcement

3. Endpoint'ler:

GET /api/compliance/checklists — Tüm checklist'ler (filtre: vesselId, standard)
POST /api/compliance/checklists — Yeni checklist oluştur (standart seç → otomatik maddeler)
GET /api/compliance/checklists/:id — Checklist detay + tüm maddeler
PATCH /api/compliance/checklists/:id — Checklist güncelle

GET /api/compliance/checklists/:id/items — Checklist maddeleri
PATCH /api/compliance/items/:itemId — Madde güncelle (compliant/evidence/finding)
POST /api/compliance/items/:itemId/upload-evidence — Kanıt dosyası yükle

POST /api/compliance/checklists/:id/audits — Denetim kaydı ekle
GET /api/compliance/checklists/:id/audits — Denetim geçmişi

GET /api/compliance/dashboard — Genel uyumluluk durumu (tüm gemiler, tüm standartlar)
GET /api/compliance/vessels/:vesselId/status — Gemi bazlı uyumluluk durumu
GET /api/compliance/expiring — Yaklaşan denetim tarihleri

4. client/src/pages/compliance.tsx sayfası oluştur:

   Genel Bakış:
   - Uyumluluk skoru kartları: ISM %92, ISPS %85, MLC %78
   - Daire grafik (radial progress) ile her standart
   - Toplam açık bulgular sayısı (kırmızı badge)
   - Yaklaşan denetim tarihleri listesi

   Gemi Bazlı Görünüm:
   - Gemi seç → o geminin tüm standartlardaki durumu
   - Her standart bir kart: ilerleme çubuğu + bulgu sayısı + son denetim

   Checklist Detay:
   - Section bazlı accordion
   - Her madde: checkbox (uygun/değil) + kanıt açıklaması + dosya yükleme
   - Bulgu ekleme: gözlem / uygunsuzluk / majör uygunsuzluk
   - Düzeltici faaliyet: açıklama + sorumlu + vade + durum
   - İlerleme çubuğu: %X tamamlandı

   Denetim Kaydı:
   - Denetim ekle formu: tip, denetçi, tarih, genel sonuç
   - Bulgu listesi: madde seç + bulgu ekle
   - Denetim raporu dosyası yükleme

   Düzeltici Faaliyetler:
   - Tüm açık düzeltici faaliyetler tablosu
   - Vade tarihi geçmiş olanlar kırmızı
   - Filtre: gemi, standart, durum, sorumlu
