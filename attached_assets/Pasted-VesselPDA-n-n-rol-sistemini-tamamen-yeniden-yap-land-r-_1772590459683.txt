VesselPDA'nın rol sistemini tamamen yeniden yapılandır. Şu an 3 rol var ve admin kayıt sayfasında seçilebiliyor — bunu değiştir.

YENİ ROL SİSTEMİ: 5 rol olacak: ship_agent, shipowner, ship_broker, ship_provider ve admin.
Admin rolü kayıt sayfasında GÖRÜNMEYECEK — sadece veritabanından manuel atanabilir.

Yapılacaklar:

1. shared/schema.ts — users tablosundaki role alanını güncelle:
   role enum değerleri: "ship_agent", "shipowner", "ship_broker", "ship_provider", "admin"
   
   Eski roller varsa migration ile eşle:
   - "agent" → "ship_agent"
   - "owner" → "shipowner"
   - "broker" → "ship_broker"
   - "provider" → "ship_provider"
   - "admin" → "admin" (aynen kalır)
   
   Eğer mevcut role enum farklı isimler kullanıyorsa (örneğin "ship-agent", "ship_owner" vs.) onları da yeni isimlere eşle.

2. server/middleware/role-guard.ts oluştur:

   requireRole(...roles: string[]) middleware fonksiyonu:
   - req.user.role kontrol eder
   - İzin verilen rollerden biriyse next() çağır
   - Değilse 403 döndür: { message: "Bu işlem için yetkiniz yok" }
   - Admin rolü HER ZAMAN geçer (admin tüm endpoint'lere erişebilir)
   
   export function requireRole(...roles: string[]) {
     return (req, res, next) => {
       if (!req.user) return res.status(401).json({ message: "Giriş yapmalısınız" });
       if (req.user.role === "admin") return next();
       if (roles.includes(req.user.role)) return next();
       return res.status(403).json({ message: "Bu işlem için yetkiniz yok" });
     };
   }
