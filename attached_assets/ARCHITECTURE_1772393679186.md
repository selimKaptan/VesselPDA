# Proforma Disbursement Account (PDA) Sistemi

## Mimari Doküman & Teknik Rehber

---

## 1. Genel Bakış

Bu sistem, gemi acentelerinin liman masraf proformalarını otomatik olarak hesaplamasını sağlar.
Gemi bilgileri girildiğinde, seçilen liman/terminal için tüm masraf kalemlerini resmi tarifelere göre hesaplar
ve profesyonel bir PDA (Port Disbursement Account) dokümanı üretir.

### Teknoloji Stack
- **Frontend:** Next.js 14+ (App Router) + React + TypeScript
- **Backend:** Next.js API Routes veya Express.js
- **Veritabanı:** PostgreSQL
- **ORM:** Prisma
- **Döviz Kuru:** TCMB API veya Google Finance

## 5. Tarife Güncelleme Rehberi

### Senaryo 1: Kıyı Emniyeti Tarifesi Değişti
```
1. tariffs/kiyi-emniyeti/pilotage-tugboat-mooring.json dosyasını aç
2. İlgili "tiers" altındaki base_rate_usd ve per_1000_grt_usd değerlerini güncelle
3. Admin panelinden veya API ile DB'deki tariff_rate_tiers tablosunu güncelle
4. effective_from tarihini yeni tarife başlangıcına ayarla
```

### Senaryo 2: Yeni Liman/Terminal Ekleme
```
1. port-configs/terminal-configs.json'da _template'i kopyala
2. Terminal kodunu, adını, aktif kalemleri ve sabit masrafları tanımla
3. DB'ye ports ve terminals tablosuna kayıt ekle
4. İlgili tariff_items ve tariff_rate_tiers kayıtlarını oluştur
```

### Senaryo 3: Yeni Bir Masraf Kalemi Ekleme
```
1. tariff_categories tablosuna yeni kategori ekle (ör: 'FUEL_SURCHARGE')
2. İlgili JSON dosyasına hesaplama kuralını ekle
3. Hesaplama motoruna yeni bir fonksiyon ekle
4. Terminal config'lerine yeni kalemi ekle (enabled: true/false)
```

### Senaryo 4: Döviz Kuru Güncelleme
```
- Otomatik: TCMB API veya Google Finance'den günlük çekilir
- Manuel: Admin panelinden veya exchange_rates tablosuna eklenir
- Proforma anında o günkü kur otomatik kullanılır
```

## 8. Yeni Liman Ekleme Adımları (Özet)

1. **`ports` tablosuna** yeni liman ekle
2. **`terminals` tablosuna** terminal ekle (has_pilotage, has_vts vb. belirle)
3. **`terminal-configs.json`'a** terminal bloğu ekle (aktif kalemler + sabit masraflar)
4. **`tariff_items` tablosuna** her aktif kalem için kayıt oluştur
5. **`tariff_rate_tiers` tablosuna** kademe verilerini gir
6. **`terminal_parameters` tablosuna** sabit parametreleri gir
7. **Test et:** Bilinen bir gemi ile hesaplama yap, Excel ile karşılaştır

## 9. Dikkat Edilecek Noktalar

### Döviz & Yuvarlama
- TL cinsinden hesaplanan kalemler (sağlık, gümrük, DTO, LÇB) → USD'ye çevrilir (/ USD_TRY)
- EUR cinsinden kalemler (acente, süpervizyon) → USD'ye çevrilir (× EUR/USD)
- EUR sütunu = USD tutarı / EUR/USD paritesi
- ROUNDUP kullanılan yerler: GRT/1000, NRT/1000, barınma birimi

### Özel Kurallar
- **Tehlikeli yük:** Kılavuzluk, römorkör, palamar → %30 ilave (×1.3)
- **Römorkör sayısı:** GRT > 5000 → 4 adet, değilse 2 adet
- **Barınma (TCDD İzmir):** Farklı hesap formülü
- **Gümrük (Tekirdağ):** Sabit 7900 TL
- **ISS Palm:** Acente ücreti sabit $2400, daha az kalem
- **Acente ek gün:** 7 günü aşan her 5 gün için %20 ilave

### Performans
- Tarife verileri değişmediği sürece cache'lenebilir
- Döviz kuru günde 1 kez çekilir
- Proforma hesaplama < 100ms olmalı (tüm lookup'lar indexed)
