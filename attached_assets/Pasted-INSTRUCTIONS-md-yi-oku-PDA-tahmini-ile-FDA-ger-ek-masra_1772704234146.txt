INSTRUCTIONS.md'yi oku. PDA (tahmini) ile FDA (gerçek) masrafları karşılaştıran profesyonel rapor sayfası ekle. Bu rapor agent'ın tahmin doğruluğunu gösterir ve gelecek proformalar için referans olur.

=== 1. API ENDPOINT ===

server/routes/ içinde uygun modüle ekle:

GET /api/da-comparison/:proformaId
- isAuthenticated
- Proforma ve bağlı FDA'yı çek
- Line item bazında karşılaştırma oluştur
- Response:

{
  proforma: { id, referenceNumber, vesselName, portName, totalUsd, totalEur, createdAt },
  fda: { id, referenceNumber, totalActualUsd, totalActualEur, approvedAt },
  comparison: [
    {
      description: "Pilotage (Inward)",
      category: "Pilotage",
      estimatedUsd: 2850,
      actualUsd: 2920,
      varianceUsd: 70,
      variancePercent: 2.46,
      status: "over" // "over" | "under" | "exact"
    },
    ...
  ],
  summary: {
    totalEstimatedUsd: 27340,
    totalActualUsd: 28150,
    totalVarianceUsd: 810,
    totalVariancePercent: 2.96,
    itemsOver: 5,    // bütçeyi aşan kalem sayısı
    itemsUnder: 7,   // bütçe altında kalem sayısı
    itemsExact: 2,   // tam isabet
    accuracyScore: 97.04  // 100 - variancePercent
  }
}

Logic:
- Proforma'nın lineItems ve FDA'nın lineItems'ını description ile eşleştir
- Eşleşmeyen kalemleri "New Item" veya "Removed Item" olarak işaretle
- Variance = actual - estimated
- Pozitif variance = bütçe aşımı (over)
- Negatif variance = tasarruf (under)
- %1'den az fark = exact

GET /api/da-comparison/history?limit=10
- isAuthenticated
- Kullanıcının tüm PDA-FDA çiftlerini listele
- Her birinin accuracy score'u ile
- Agent'ın genel tahmin doğruluğu ortalaması

=== 2. FRONTEND — DA COMPARISON PAGE ===

client/src/pages/da-comparison.tsx oluştur:

Sayfa 2 modda çalışır:

MODE 1: Tek Karşılaştırma (URL: /da-comparison/:proformaId)
MODE 2: Geçmiş Listesi (URL: /da-comparison)

--- MODE 2: LİSTE ---

Üstte:
- "DA Comparison" başlık
- Genel accuracy score kartı: büyük yuvarlak gösterge
  * Yeşil daire: "%97.2 Average Accuracy"
  * Alt text: "Based on 8 completed disbursements"

Tablo: tüm PDA-FDA çiftleri
- PDA Ref | Vessel | Port | Estimated | Actual | Variance | Accuracy | Date
- Her satır tıklanınca detaya git
- Accuracy sütunu renk kodlu:
  * >95%: yeşil
  * 90-95%: amber
  * <90%: kırmızı

--- MODE 1: DETAY ---

Üstte özet kartları (4'lü grid):

Kart 1: "Estimated Total"
  $27,340 — mavi arka plan

Kart 2: "Actual Total"
  $28,150 — yeşil arka plan

Kart 3: "Total Variance"
  +$810 (+2.96%) — kırmızı veya yeşil

Kart 4: "Accuracy Score"
  97.04% — büyük, yuvarlak progress gösterge
  Yeşil/amber/kırmızı renk

Orta: Detaylı karşılaştırma tablosu

Tablo header: # | Description | Estimated USD | Actual USD | Variance USD | Variance % | Status
- Her satır:
  * Over budget: kırmızı arka plan hafif, ↑ ok
  * Under budget: yeşil arka plan hafif, ↓ ok
  * Exact: normal
- Variance % sütunu: renk kodlu bar (inline mini bar chart)
- "New Item" badge: FDA'da var PDA'da yok
- "Removed" badge: PDA'da var FDA'da yok

Alt: Variance bar chart (Recharts):
- Yatay bar chart
- Her line item bir bar
- Pozitif (kırmızı, sağa) ve negatif (yeşil, sola)
- Ortada 0 çizgisi

Sağ panel: Insights kartı
- "Top Overestimates" — en çok fark olan 3 kalem (fazla tahmin)
- "Top Underestimates" — en az tahmin edilen 3 kalem
- "Suggestions" — otomatik öneriler:
  * "Pilotage fees were consistently underestimated by 5%. Consider adjusting your tariff rates."
  * "Berthing costs were overestimated by 12%. You may be using outdated daily rates."
  * "Your overall accuracy of 97% is excellent."

Alt: İki belge linki
- "View PDA →" /proformas/:id
- "View FDA →" /fda/:fdaId

PDF Export butonu: DA Comparison raporu PDF olarak indir

=== 6. IMPORTANT ===

- Line item eşleştirme description bazında (fuzzy match değil, exact)
- Eşleşmeyen itemler ayrı gösterilsin
- Accuracy score: 100 - abs(variancePercent)
- Veri yoksa (FDA henüz oluşturulmamış): "No FDA linked to this PDA yet" mesajı
- Mevcut sayfaları bozma
- Mobile responsive
- Cache: comparison sonuçlarını medium cache'le
