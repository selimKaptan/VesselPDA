PDA APPROVAL FLOW — PDA onay akışı sistemi ekle. Mevcut kodu bozmadan yeni özellik ekle.

BÖLÜM A — DATABASE DEĞİŞİKLİKLERİ (shared/schema.ts):

1. proformas tablosuna şu alanları ekle (yoksa):
   - approvalStatus: text("approval_status").default("draft")
     Değerler: "draft", "sent", "under_review", "revision_requested", "approved", "rejected"
   - sentAt: timestamp("sent_at")
   - reviewedAt: timestamp("reviewed_at")
   - reviewedBy: integer("reviewed_by")
   - revisionNote: text("revision_note")
   - approvalNote: text("approval_note")

2. Yeni tablo: proforma_approval_logs
   - id: serial primary key
   - proformaId: integer → proformas FK
   - userId: integer → users FK
   - action: text (sent, reviewed, revision_requested, approved, rejected)
   - note: text (nullable)
   - previousStatus: text
   - newStatus: text
   - createdAt: timestamp default now()

3. drizzle-kit push yap

BÖLÜM B — BACKEND API:

1. server/routes/proforma-approval.ts oluştur:

   POST /api/proformas/:id/send
   - Sadece proforma sahibi (agent) yapabilir
   - Status: draft → sent
   - Log kaydı oluştur
   - Response: güncel proforma

   POST /api/proformas/:id/review
   - Sadece shipowner veya admin yapabilir
   - Body: { action: "approve" | "reject" | "request_revision", note?: string }
   - Status geçişleri:
     * approve: sent/under_review → approved
     * reject: sent/under_review → rejected
     * request_revision: sent/under_review → revision_requested
   - Log kaydı oluştur
   - Response: güncel proforma

   GET /api/proformas/:id/approval-history
   - Proforma'nın onay geçmişi (log kayıtları)
   - Her log: kullanıcı adı, aksiyon, tarih, not

   GET /api/proformas/pending-approval
   - Sadece shipowner/admin: onay bekleyen PDA'lar listesi
   - Status: sent veya under_review olanlar
   - Sıralama: en eski önce

2. server/routes/index.ts'te yeni route'u register et
