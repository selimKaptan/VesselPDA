Deploy hatası: "The deployment is failing health checks. Health checks are sent to the / endpoint and must respond as soon as possible. Make sure that the / endpoint returns a 200 status code in a timely manner."

İki kritik sorun var. İkisini de düzelt:

SORUN 1: "/" endpoint'i yok veya frontend serve edilmiyor.

server/index.ts (veya ana server dosyası — server/main.ts, server/app.ts hangisi varsa) dosyasında, TÜM API route'larından SONRA ama app.listen'dan ÖNCE şunu ekle:

// Health check endpoint - Replit deployment için ŞART
app.get("/api/health", (req, res) => {
  res.status(200).json({ status: "ok" });
});

// Production'da frontend static dosyalarını sun
const distPath = path.resolve(process.cwd(), "dist/public");
// dist/public yoksa dist/ dene
const fs = require("fs");
const actualDistPath = fs.existsSync(distPath) ? distPath : path.resolve(process.cwd(), "dist");

app.use(express.static(actualDistPath));

// SPA fallback - API olmayan tüm istekleri index.html'e yönlendir
app.get("*", (req, res) => {
  if (req.path.startsWith("/api")) {
    return res.status(404).json({ message: "API endpoint not found" });
  }
  const indexPath = path.join(actualDistPath, "index.html");
  if (fs.existsSync(indexPath)) {
    res.sendFile(indexPath);
  } else {
    res.status(200).send("VesselPDA is running");
  }
});

ÖNEMLİ: path ve fs import'larını dosyanın üstüne ekle:
import path from "path";
import fs from "fs";

SORUN 2: AIS ve background job'lar server başlangıcını engelliyor.

Server başlangıç sırası MUTLAKA şöyle olmalı:

a) Express app oluştur + middleware ekle
b) TÜM route'ları bağla
c) Static dosyaları sun
d) app.listen → "Server ready" (BU AN health check'e 200 dönebilmeli)
e) 30 saniye SONRA: AISStream başlat (setTimeout içinde, try-catch ile)
f) 60 saniye SONRA: Background job'lar — geocoding, sanctions fetch (setTimeout içinde, try-catch ile)

AISStream başlatan kodu bul. Muhtemelen şunlardan birinde:
- server/ais.ts
- server/services/ais-stream.ts
- server/services/ais.ts
- server/index.ts içinde direkt

Bu kodu şöyle değiştir:
- Server başlangıcında ÇAĞIRMA
- setTimeout(() => { try { startAIS(); } catch(e) { console.log("AIS optional, skipping"); } }, 30000);

Geocoding başlatan kodu bul ("Starting background geocoding for 917 ports"):
- Bu da server başlangıcında ÇAĞRILMASIN
- setTimeout(() => { try { startGeocoding(); } catch(e) { console.log("Geocoding optional, skipping"); } }, 60000);

Sanctions fetch kodu ("Fetching OFAC SDN list"):
- Bu da geciktirilsin
- setTimeout(() => { try { fetchSanctions(); } catch(e) { console.log("Sanctions optional, skipping"); } }, 45000);

KONTROL: Değişikliklerden sonra server başladığında console'da şu sıra görünmeli:
1. "Server is ready on port 5000" (HEMEN — 1-2 saniye içinde)
2. ... 30 saniye sessizlik ...
3. "AISStream: connecting..." (30 saniye sonra)
4. "Starting background geocoding..." (60 saniye sonra)

Server HEMEN ayağa kalkmalı. Ağır işler SONRA başlamalı.
