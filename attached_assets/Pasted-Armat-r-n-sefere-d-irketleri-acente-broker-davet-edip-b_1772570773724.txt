Armatörün sefere dış şirketleri (acente, broker) davet edip birlikte çalıştığı işbirliği sistemi ekle.

Yapılacaklar:

1. shared/schema.ts dosyasına şu tabloyu ekle:

voyageCollaborators tablosu:
- id (serial PK)
- voyageId (integer, FK voyages, not null)
- organizationId (integer, FK organizations, nullable) — davet edilen şirket
- userId (varchar, FK users, nullable) — bireysel davet
- invitedByUserId (varchar, FK users, not null) — davet eden
- role (text, default "viewer") — "manager", "editor", "viewer"
- permissions (jsonb, not null) — sefer bazlı özel yetkiler:
  {
    "viewVoyage": true,
    "editVoyage": false,
    "viewDocuments": true,
    "uploadDocuments": true,
    "deleteDocuments": false,
    "signDocuments": false,
    "viewProforma": true,
    "editProforma": false,
    "viewInvoice": true,
    "viewChecklist": true,
    "editChecklist": false,
    "viewChat": true,
    "sendChat": true,
    "viewTimeline": true
  }
- status (text, default "pending") — pending, accepted, declined
- invitedAt (timestamp, default now)
- respondedAt (timestamp)
- notes (text, nullable)

2. Endpoint'ler (server/routes/voyages.ts'e ekle):

POST /api/voyages/:id/collaborators — Sefere işbirlikçi davet et
  body: { organizationId veya userId, role, permissions, notes }
  Davet edilene bildirim + email gönder

GET /api/voyages/:id/collaborators — Sefer işbirlikçileri listesi

PATCH /api/voyages/:id/collaborators/:collabId — İşbirlikçi yetkilerini güncelle
  Sefer sahibi veya manager rolü gerekli

DELETE /api/voyages/:id/collaborators/:collabId — İşbirlikçiyi kaldır

PATCH /api/voyages/:id/collaborators/:collabId/respond — Daveti kabul/reddet
  body: { status: "accepted" veya "declined" }

GET /api/voyages/shared — Bana paylaşılan seferler listesi
  Kullanıcının davet edildiği tüm seferleri getir

3. Voyage erişim kontrolünü güncelle:
   Mevcut: Sadece sefer sahibi görebilir
   Yeni: Sefer sahibi + aynı organizasyon üyeleri + davet edilen collaborator'lar
   
   Her voyage endpoint'inde erişim kontrolü:
   - canAccessVoyage(userId, voyageId): sefer sahibi mi, aynı org mu, collaborator mı?
   - canEditVoyage(userId, voyageId): sahibi mi, org'da edit yetkisi var mı, collaborator olarak editor/manager mı?
   - checkVoyagePermission(userId, voyageId, permission): collaborator'un belirli yetkisi var mı?

4. client/src/pages/voyage-detail.tsx güncelle:
   - "İşbirlikçiler" tab'ı ekle
   - İşbirlikçi ekleme dialog'u:
     • Organizasyon veya kullanıcı arama
     • Rol seçimi (Manager, Editor, Viewer)
     • Yetki ayarlama: her özellik için on/off toggle
       - Seferi görüntüle / düzenle
       - Dokümanları görüntüle / yükle / sil / imzala
       - Proforma görüntüle / düzenle
       - Fatura görüntüle
       - Checklist görüntüle / düzenle
       - Chat görüntüle / mesaj gönder
       - Timeline görüntüle
   - İşbirlikçi kartları: şirket adı/logo, rol, yetki özeti, davet durumu
   - Yetkiyi düzenle / kaldır butonları

5. Paylaşılan seferler listesi:
   - client/src/pages/voyages.tsx sayfasına "Paylaşılan Seferler" tab'ı ekle
   - Bana paylaşılan tüm seferler listesi
   - Bekleyen davetler (kabul et / reddet)

6. Voyage sayfasında yetki bazlı UI:
   - Yetkisi olmayan butonları gizle veya disable et
   - Örnek: uploadDocuments=false ise "Doküman Yükle" butonu görünmesin
   - Örnek: editChecklist=false ise checklist sadece readonly gösterilsin

Bildirim gönder: Davet edildiğinde, kabul/red edildiğinde, yetki değiştiğinde.
